704.二分查找

给定一个n个元素有序的（升序）整型数组nums和一个目标值target，写一个函数搜索nums中的target，如果目标值存在返回下标，否则返回-1。

要点

  1. 这道题目的前提是数组为有序数组，同时题目还强调数组中无重复元素

  2. 定义在左闭右闭的[left, right]的区间里

  3. while (left <= right) 要使用 <= ，因为 left == right 是有意义的，所以使用 <=

27.移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并 原地 修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

要点

  1. 这个题目暴力的解法就是两层for循环，一个for循环遍历数组元素 ，第二个for循环更新数组
  2. 推荐使用双指针法，通过一个快指针和慢指针在一个for循环下完成两个for循环的工作
  3. 定义快慢指针
    • 快指针：寻找新数组的元素 ，新数组就是不含有目标元素的数组
    • 慢指针：指向更新 新数组下标的位置
  4. 双指针法（快慢指针法）在数组和链表的操作中是非常常见的，很多考察数组、链表、字符串等操作的面试题，都使用双指针法

代码

心得

只能想到暴力破解法😂，但是学习到了双指针法还是挺有收获的。

977.有序数组的平方

给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

要点

  1. 暴力排序 最直观的想法，每个数平方之后，排个序。排序代码为

  2. 双指针法 数组其实是有序的，只不过负数平方之后可能成为最大数了。那么数组平方的最大值就在数组的两端，不是最左边就是最右边，不可能是中间。此时可以考虑双指针法了，start指向起始位置，end指向终止位置。此时的时间复杂度为O(n)，相对于暴力排序的解法O(n + nlog n)还是提升不少的

心得

  1. 这题一开始踩了很多坑，包括初始化新数组的方式、排序函数的引用。最后只能想到使用冒泡排序来暴力破解，实在是效率太低。
  2. 后面看了解题思路后豁然开朗，在复现代码时不停出现地址溢出的报错，查询之后才发现C++不支持对多个变量的连续赋值，例如int end,index=0。平时python养成的坏习惯，要改正呀。
  3. 对于vector变量的初始化：
    1. 使用默认构造函数，创建一个空的vector，例如：vector<int> v;。这时候v的size为0，如果直接访问v[i]会报错。这种方式适用于元素个数未知，需要动态添加的情况。
    2. 使用拷贝构造函数，使用已有的vector容器来初始化另一个vector容器，两者内容完全相同，例如：vector<int> v1(v);或者vector<int> v1 = v;。这种方式适用于需要复制一个vector的情况。
    3. 使用列表初始化，使用花括号括起来的一组值来初始化vector，例如：vector<int> v = {1, 2, 3, 4, 5};。这时候v的size为5，v[i]为花括号中的第i+1个值。这种方式适用于已知元素值的情况。
    4. 使用范围初始化，使用两个迭代器指定的范围中的元素来初始化vector，例如：vector<int> v(arr, arr + 5);。这时候v的size为5，v[i]为arr[i]。这种方式适用于从数组或其他容器中复制一部分元素的情况。
    5. 使用大小初始化，使用一个整数n来指定vector的大小，例如：vector<int> v(10);。这时候v的size为10，每个元素初始化为默认值0。这种方式适用于已知元素个数，但不知道元素值的情况。
    6. 使用大小和值初始化，使用一个整数n和一个值m来指定vector的大小和初始值，例如：vector<int> v(10, 1);。这时候v的size为10，每个元素初始化为1。这种方式适用于已知元素个数和值的情况。

209. 长度最小的子数组

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其总和大于等于 target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

要点

  1. 滑动窗口 就是不断的调节子序列的起始位置和终止位置，从而得出我们要想的结果
  2. 首先要思考 如果用一个for循环，那么应该表示滑动窗口的起始位置，还是终止位置。如果只用一个for循环来表示滑动窗口的起始位置，那么如何遍历剩下的终止位置此时难免再次陷入 暴力解法的怪圈。所以只用一个for循环，那么这个循环的索引，一定是表示滑动窗口的终止位置
  3. 滑动窗口解题：
    1. 窗口内是什么？
    2. 如何移动窗口的起始位置？
    3. 如何移动窗口的结束位置？
  4. 窗口就是 满足其和 ≥ target 的长度最小的连续子数组。窗口的起始位置如何移动：如果当前窗口的值大于 target了，窗口就要向前移动了（也就是该缩小了）。窗口的结束位置如何移动：窗口的结束位置就是遍历数组的指针，也就是for循环里的索引
  5. 滑动窗口的精妙之处在于根据当前子序列和大小的情况，不断调节子序列的起始位置。从而将O(n^2)暴力解法降为O(n)
